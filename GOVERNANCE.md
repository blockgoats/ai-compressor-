# Governance

This document describes **how decisions are made** in this repository. It is lightweight by design and can grow with the project.

## Maintainers

**Maintainers** merge pull requests, cut releases, and enforce the [Code of Conduct](CODE_OF_CONDUCT.md). The list of people with merge access is defined on GitHub (organization / repository settings), not necessarily in this file.

If you need a maintainer decision, open an issue and @-mention maintainers or use a discussion thread if enabled.

## Contributions

- **Code and docs:** Submitted via **pull request** against the default branch (usually `main`), reviewed by maintainers or trusted contributors.
- **Small fixes:** Typos, obvious bugfixes, and documentation improvements are welcome without a prior issue when scope is clear.
- **Large features:** Especially embedding-level RCP work, should align with [docs/roadmap.md](docs/roadmap.md) and often benefit from an **issue first** to agree on direction.

## Releases

See [RELEASING.md](RELEASING.md) for versioning and tagging expectations. [CHANGELOG.md](CHANGELOG.md) should reflect user-visible changes.

## Changes to governance

Updates to this file should be proposed via pull request so the community can see the diff.
